## Design Principles

These principles guide every design decision across the CorpAI platform:

- **One dominant action per screen** — Every view has a single primary action. Secondary actions are visually de-emphasized.
- **Whitespace over borders** — Separate content with space, not lines. Borders are reserved for interactive states and focus indicators.
- **Motion indicates intelligence** — Animation is not decorative. It signals AI activity, state transitions, and system feedback.
- **Color indicates meaning, not decoration** — Every color token carries semantic weight. Never use color purely for visual variety.
- **Information first** — Content density adapts to the task. Dashboards may be dense; reading views are spacious.
- **Density adapts to expertise** — Novice users see generous spacing and guidance. Power users can opt into compact layouts.

## Brand & Style

This design system embodies the intersection of high-stakes corporate reliability and cutting-edge artificial intelligence. The aesthetic is rooted in **Minimal Optical Glass** — a restrained, precision-focused approach to translucency — with a focus on "Intelligence in the Light": transitioning from a dark environment to a clean, high-clarity professional interface utilizing light surfaces contrasted by vibrant, energetic pulses of green and teal.

The target audience consists of enterprise decision-makers and power users who require a tool that feels both authoritative and futuristic. The UI avoids the clutter of traditional "boxed" layouts, opting instead for an expansive, edge-to-edge cinematic experience that prioritizes data visualization and agent-driven interactions. Every element should feel like a precision instrument: sharp, responsive, and illuminated with clarity.

### Glass Usage Guidelines

Glass effects are used as an **accent**, not as the primary surface treatment:

| Property | Value |
|----------|-------|
| Glass opacity | 5–10% |
| Blur | 12–16px |
| Border | 1px inner glow at 10% opacity |
| Application | Overlays, modals, floating panels only |

Avoid glass on content-heavy surfaces (tables, lists, forms) to preserve readability and reduce GPU load.

## Elevation & Depth

Depth is created through **Optical Translucency** and light stacking rather than traditional dark shadows.

1.  **Base Layer:** The pure white "Neutral" background (#FFFFFF).
2.  **Glass Layer:** Semi-transparent white surfaces with a `backdrop-filter: blur(12px)` and a 1px "inner glow" border (Sber Green at 10% opacity).
3.  **Active Elevation:** Elements that require focus are highlighted with a soft **Emerald or Teal Glow** shadow — very large blur (40px+), very low opacity (10%).
4.  **Floating Elements:** Use "ghost borders" — ultra-thin, low-contrast lines that define shape without closing the layout.

## Workspace Model

CorpAI is designed as an **AI Operating System** — a persistent workspace for interacting with intelligent agents, not a collection of pages. The interface is organized into five distinct layers:

### Navigation Layer
Global navigation and workspace switching.

- Persistent sidebar (collapsible, 240px / 64px collapsed)
- Workspace switcher (personal, team, organization)
- Agent roster — list of available agents with status indicators
- Quick-access command palette (`Cmd+K`)

### Conversation Layer
Chat, history, and agent management.

- Thread list (left panel, resizable)
- Active conversation area (center)
- Agent context bar — shows which agent is active, its tools, and sources
- Message composer with multi-modal input (text, files, voice)
- Streaming token display with AI motion tokens

### Canvas Layer
Documents, diagrams, code, and analytics.

- Full-width canvas area for rich artifacts
- Tabbed document interface
- Embedded chart, diagram, and formula renderers
- Split-view for side-by-side code and preview
- Zoom and pan controls

### Context Layer
Properties, data sources, and tools.

- Right panel (collapsible, 320px)
- Source attribution — which documents/knowledge bases informed the response
- Tool call log — transparent record of agent tool usage
- Confidence indicators and citations
- Configuration panel for active agent

### Transient Layer
Notifications, command palette, and modals.

- Toast notifications (top-right, auto-dismiss)
- Command palette (overlay, full-screen dim)
- Modal dialogs for confirmations and forms
- Floating tooltips and context menus
- Agent suggestion cards (peek from bottom)

### Glow-Lines
Use horizontal 1px lines that fade to transparent at the ends to separate large content blocks, suggesting a "scanning" or "laser" aesthetic.

## Accessibility

CorpAI is committed to enterprise-grade accessibility. All interfaces must meet **WCAG 2.1 AA** as a minimum.

### Contrast

- **Minimum contrast ratio:** 4.5:1 for normal text, 3:1 for large text (18px+ / 14px bold+)
- **Interactive elements:** 3:1 minimum against adjacent colors
- **Focus indicators:** 3:1 against the component surface

### Keyboard

- **Full keyboard navigation:** All interactive elements reachable via Tab
- **Visible focus:** Every focusable element shows a clear focus ring (`shadow-focus`)
- **Focus order:** Logical DOM order matching visual layout
- **Skip link:** "Skip to content" as the first focusable element

### Reduced Motion

- Respect the `prefers-reduced-motion` OS setting
- When enabled: disable all non-essential animations (pulse, scan, gradient-shift)
- Essential animations only: loading spinners, progress bars, streaming text

### Screen Readers

- All icons use `aria-hidden="true"` with descriptive text in adjacent elements
- Dynamic content (streaming, tool calls) announces changes via `aria-live="polite"`
- Error states include `role="alert"`
- Landmark regions (`<nav>`, `<main>`, `<aside>`) used for all workspace layers
