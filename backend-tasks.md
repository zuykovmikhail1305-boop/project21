### 1. Проектирование БД и интеграция с хранилищем (Data Layer)

* 
**Интеграция со СберДиском / S3-хранилищем:** Реализация сервиса взаимодействия с API облака для выкачивания файлов, отслеживания изменений (через вебхуки или поллинг) и синхронизации структуры каталогов.


* **Проектирование реляционной БД (PostgreSQL):** Создание схем для хранения истории чатов, сессий пользователей, метаданных документов и, главное, маппинга прав доступа (Active Directory / ACL).
* 
**Настройка Векторной БД (Qdrant / Milvus / pgvector):** Развертывание базы данных, проектирование коллекций для векторных представлений (эмбеддингов) текстовых чанков.



### 2. Реализация асинхронного пайплайна обработки данных (ETL / Document Processing)

* **Асинхронная очередь задач (Celery + Redis):** Разработка архитектуры обработки «тяжелых» документов (длинных PDF, Excel) в фоновом режиме, чтобы не блокировать API-сервер.
* **Умный парсинг и чанкинг (Парсер документов):** * Реализация текстового разбиения (Chunking) документов с сохранением метаданных.
* *Специфическая задача кейса:* Написание парсеров для конвертации таблиц **XLSX** в Markdown/HTML, а также обработка многоколоночных **PDF** (с использованием инструментов вроде *LlamaParse* или *LayoutLM*), чтобы не ломать пространственную связь данных.


* 
**Генерация эмбеддингов:** Написание микросервиса, который берет готовые чанки текста, отправляет их в модель эмбеддингов и сохраняет векторы вместе с жестко привязанными метаданными.



### 3. Инфраструктура безопасности и прав доступа (Критический вызов)

* **Реализация Динамического ACL (Pre-filtering):** Ваша главная задача в сфере ИБ — сделать так, чтобы AI-агент не выдал конфиденциальную информацию пользователю без прав. Вам нужно будет:
1. При парсинге файла записывать в метаданные вектора разрешенные ID групп доступа.
2. Извлекать группы доступа текущего пользователя из его JWT-токена авторизации.
3. Дописывать логику фильтрации прямо в векторный запрос к DB (запрос должен искать только по тем чанкам, где группы пользователя совпадают с группами в метаданных).



### 4. Разработка ядра ИИ и логики оркестрации (Agentic RAG)

* 
**Семантический роутер (Semantic Router):** Создание бэкенд-логики, которая с помощью легких классификаторов или LLM-промптов определяет намерение пользователя и направляет запрос в нужный индекс (поиск по регламентам, отчетам или презентациям).


* 
**Агентская оркестрация (LangGraph / LangChain):** Написание сценариев (графов) взаимодействия агентов. Описание логики работы: *Router Agent $\rightarrow$ Search/RAG Agent $\rightarrow$ Summarizer Agent*.


* 
**Интеграция LLM (GigaChat API или локальные LLM):** Подключение к API моделей, управление контекстным окном, настройка системных промптов для соблюдения правила *Strict Grounding* (модель отвечает строго по контексту, без галлюцинаций).


* 
**Переранжирование (Reranking):** Написание сервиса интеграции с Cross-Encoder моделями для точного отбора `top-k` контекстов перед отправкой финального промпта в LLM.



### 5. API, Валидация и логика вывода

* 
**Разработка REST API (FastAPI):** Создание эндпоинтов для взаимодействия с фронтендом: авторизация, отправка сообщений в чат, получение истории, получение структуры документов.


* **Валидационный слой (Validation Layer / NLI):** Реализация автоматического факт-чекинга. Написание скрипта, который берет сгенерированный ответ LLM, берет исходные чанки из БД и проверяет их на логическое соответствие (модель-арбитр).


* **Логика цитирования (Citations):** Формирование ответа бэкенда в виде структурированного JSON, содержащего текст ответа и массив `chunk_id` / `source_links`. Это позволит фронтенду подсвечивать страницы файлов, на которые ссылается агент.
* **Стриминг ответов:** Настройка потоковой передачи текстового ответа от LLM пользователю (Server-Sent Events) для плавного UX в интерфейсе.

### 6. DevOps и развертывание

* 
**Контейнеризация (Docker / Docker Compose):** Сборка всех компонентов (FastAPI, Celery workers, Redis, Qdrant/PostgreSQL) в изолированные контейнеры для быстрой демонстрации MVP и потенциального On-Premise развертывания в контуре Сбера.
